# UsageTracker v0.2.0 路线图

> **创建日期**: 2026-04-16  
> **状态**: 规划中（v0.1.0-beta 完成后启动）  
> **前置版本**: v0.1.0-beta

---

## 概述

v0.2.0 聚焦于**用户体验升级**和**智能化增强**，在 v0.1.0-beta 稳定运行的基础上，引入事件驱动追踪、自动更新、数据同步等高级特性。

---

## 一、核心升级项

### 1.1 事件驱动追踪（替代 5 秒轮询）

**现状**：`SetWinEventHook` 事件驱动方案在 v0.1.0 中因复杂度高被暂缓。

**v0.2.0 方案**：
- 引入 Windows `SetWinEventHook` 监听 `EVENT_SYSTEM_FOREGROUND` 和 `EVENT_OBJECT_STATECHANGE`
- 实现零延迟窗口切换检测，消除 5 秒误差
- 保留轮询作为降级方案（UWP 应用事件不稳定时自动回退）
- 新增设置项："检测模式"（自动/轮询/事件驱动）

**收益**：
- 短窗口切换（< 5s）不再丢失
- CPU 开销从定时唤醒降为纯事件驱动

### 1.2 自动更新机制

**方案**：
- 程序启动时检查固定 URL（GitHub Releases / 自建服务器）的 `version.json`
- 有新版本时托盘通知提示用户
- 支持"静默更新"和"手动下载"两种模式
- 更新包签名验证（SHA256 校验）

**配置项**：
- 自动检查更新开关（默认开）
- 检查频率：每次启动 / 每天 / 每周

### 1.3 代码签名

**目的**：消除 Windows SmartScreen 拦截警告

**方案**：
- 购买 EV 代码签名证书（约 $200-400/年）
- 对 exe 和安装包进行签名
- 签名后 SmartScreen 会自动信任，不再弹警告

**替代方案**（预算有限时）：
- 使用免费的自签名证书 + 引导用户手动信任
- 在 GitHub Release 提供 SHA256 校验值

## 二、报告增强

### 2.1 报告内翻页/筛选

**现状**：v0.1.0 的报告为静态 HTML，不支持交互筛选。

**v0.2.0 方案**：
- 日报应用明细表格增加**分类筛选器**（下拉框选择分类）
- 支持按应用名搜索
- 时长排名支持升序/降序切换
- 分页显示（每页 20 条）

### 2.2 周报/月报趋势对比

- 月报增加**同比对比**（本月 vs 上月）
- 周报增加**周环比**（本周 vs 上周）
- 趋势箭头（↑↓→）和百分比变化

### 2.3 报告导出为 PDF

- 在报告页面增加"导出 PDF"按钮
- 使用 `playwright` 或 `weasyprint` 无头浏览器渲染 HTML 后导出
- 保留图表、配色和排版

### 2.4 自定义报告时间范围

- 设置界面增加"自定义报告"功能
- 用户选择任意日期范围，生成对应报告
- 支持按分类汇总或按应用明细两种视图

---

## 三、分类系统增强

### 3.1 自动分类建议

**方案**：
- 追踪一段时间后，分析用户使用模式
- 对"其他"分类中出现频率高的应用，自动建议分类
- 例如：检测到 `code.exe` 每天出现 4 小时 → 建议"工作"分类

**实现**：
- 新增 `auto_suggest_category()` 方法
- 在设置界面"分类管理"标签页显示建议列表
- 用户一键接受或忽略

### 3.2 浏览器子分类

**方案**：
- 区分浏览器使用场景（工作浏览 vs 娱乐浏览）
- 通过窗口标题关键词推断：如标签页含"GitHub""Stack Overflow"→工作；含"YouTube""Bilibili"→娱乐
- 用户可自定义关键词规则

### 3.3 游戏时长细粒度统计

- 支持单款游戏的每日/每周/每月趋势图
- 游戏标签页增加"使用热力图"（类似 GitHub 贡献图）
- 支持设置"游戏时长目标"和超时提醒

---

## 四、反馈系统升级

### 4.1 GitHub Issues API 集成

**现状**：v0.1.0 只能"生成反馈文件"。

**v0.2.0 方案**：
- 接入 GitHub Issues API（公开仓库）
- 用户填写反馈后直接提交为 Issue
- 自动附加：版本号、操作系统、崩溃日志、系统信息（CPU/内存/Python版本）

### 4.2 使用数据匿名统计（可选）

- 用户可选择参与匿名使用统计
- 上报数据：版本号、操作系统、功能使用频率（不开机密数据）
- 用于指导后续功能优先级决策

---

## 五、性能优化

### 5.1 数据库性能

- 对大表（百万级记录）增加复合索引
- 写入策略优化：批量写入替代逐条写入
- 考虑 WAL 模式提升并发读写性能

### 5.2 内存优化

- 追踪器 sessions 列表改为循环缓冲区（只保留最近 N 条在内存）
- 历史数据完全依赖数据库，不驻留内存

### 5.3 启动速度优化

- 延迟加载非必要模块（设置界面、报告生成器在首次使用时才导入）
- 游戏扫描异步执行，不阻塞启动

---

## 六、国际化（i18n）

### 6.1 多语言支持

**v0.2.0 目标**：
- 支持简体中文、英文两种语言
- 所有 UI 文案通过语言文件管理
- 自动检测系统语言，用户可手动切换

**实现方案**：
- 使用 Python `gettext` 模块
- 语言文件：`locales/zh_CN/LC_MESSAGES/ut.po`、`locales/en/LC_MESSAGES/ut.po`
- 报告 HTML 根据语言设置注入对应文案

---

## 七、安装体验优化

### 7.1 静默安装支持

- 安装包支持 `/SILENT` 和 `/VERYSILENT` 命令行参数
- 企业部署友好

### 7.2 便携版（Portable）

- 除安装版外，提供免安装的 ZIP 便携版
- 数据默认存放在 exe 同级目录
- 适合 U 盘携带使用

---

## 八、安全加固

### 8.1 数据库加密

- 对 SQLite 数据库启用 SQLCipher 加密
- 密钥派生自 Windows DPAPI（用户级，无需输入密码）
- 防止其他程序直接读取数据库

### 8.2 进程完整性校验

- 启动时校验自身 exe 的 SHA256
- 检测是否被篡改或被 virus 替换
- 发现异常时拒绝启动并提示
