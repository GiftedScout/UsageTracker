# UsageTracker v0.3.0 版本规划

> **创建日期**: 2026-05-31  
> **状态**: 规划中  
> **前置版本**: v0.2.0-beta  
> **预估发布**: 2026 年 Q3

---

## 概述

v0.3.0 将 **v0.2.0 的 GUI 精美模式功能完整迁移到网页端**，弃用桌面 GUI 设置界面，改用本地 Web UI（Bridge 服务 `127.0.0.1:19234`）。

**核心目标**：
1. **功能迁移** — 将 v0.2.0 精美模式的所有设置功能迁移到网页端
2. **GUI 弃用** — 设置界面改为浏览器打开，tkinter 设置窗口不再维护
3. **双主题** — 网页端提供「童话」「极客」两套视觉主题
4. **无新功能** — 不新增 v0.2.0 以外的功能

---

## 一、网页端架构设计

### 1.1 技术选型

| 层级 | 方案 | 理由 |
|:-----|:-----|:-----|
| 后端 | Python 内置 `http.server` 或 `flask`（零依赖优先） | 避免新增依赖，保持精简 |
| 前端 | 原生 HTML/CSS/JS（单页应用） | 无构建步骤，直接由后端 serve |
| 通信 | Fetch API + REST 端点 | Bridge 已有 HTTP 服务，复用即可 |
| 存储 | `config.json` 读写（复用 `ConfigManager`） | 不引入新存储 |

**推荐方案**：扩展现有 `bridge_handler.py`，新增 `/settings` 端点和静态文件服务，不引入 Flask。

### 1.2 目录结构

```
ui/
  web/                    # 网页端资源（新增）
    index.html            # 主页面（SPA）
    css/
      common.css         # 通用样式
      theme-fairy.css    # 童话主题
      theme-geek.css     # 极客主题
    js/
      app.js             # 主逻辑
      api.js             # API 调用封装
      components.js       # 组件库
    assets/
      favicon.ico
      fairy/             # 童话主题图片资源
      geek/              # 极客主题图片资源
```

### 1.3 Bridge 服务扩展

现有 `bridge_handler.py` 已有 `BridgeHTTPRequestHandler`，新增端点：

| 端点 | 方法 | 功能 |
|:-----|:-----|:-----|
| `/settings` | GET | 返回设置页面 HTML |
| `/api/config` | GET | 获取当前配置（JSON） |
| `/api/config` | POST | 保存配置（JSON body） |
| `/api/apps` | GET | 获取应用分类列表 |
| `/api/apps` | POST | 新增/修改应用分类 |
| `/api/ignore` | GET/POST | 忽略名单管理 |
| `/api/games` | GET/POST | 游戏目录管理 |
| `/api/database` | GET | 数据库状态信息 |
| `/api/export-pdf` | POST | 触发 PDF 导出 |
| `/api/feedback` | POST | 提交反馈 |
| `/api/check-update` | GET | 检查更新 |
| `/static/<path>` | GET | 静态资源（CSS/JS/图片） |

## 二、功能迁移清单

以下为 v0.2.0 精美模式中已实现、需迁移到网页端的功能：

### 2.1 通用标签页（原 `ui/tab_general.py`）

| 功能 | 说明 | 迁移方式 |
|:-----|:-----|:-----|
| 应用信息展示 | 名称、版本号、许可证 | 网页 Header 区域固定显示 |
| 语言切换 | 下拉框切换 zh-CN / en | `/api/config` POST + 页面重载 |
| 开机自启开关 | Toggle 开关 | JS 调用 `/api/config` |
| 自动报告开关 | Toggle 开关 | 同上 |
| 检测模式选择 | 自动/事件驱动/轮询 | 下拉框 + `/api/config` |
| 检查更新按钮 | 手动触发 | Fetch `/api/check-update` |

### 2.2 分类管理（原 `ui/tab_categories.py`）

| 功能 | 说明 | 迁移方式 |
|:-----|:-----|:-----|
| 分类列表展示 | Treeview → HTML 表格/列表 | JS 渲染 `/api/apps` 数据 |
| 新增分类 | 对话框 → 模态弹窗 | Fetch POST `/api/apps` |
| 添加应用到分类 | 进程选择器 | 网页端实现进程列表 + 搜索 |
| 分类建议 | v0.2.0 新增功能 | 独立面板，调用后端接口 |
| 删除分类/应用 | 确认弹窗 | Fetch DELETE `/api/apps/:id` |

### 2.3 浏览器管理（原 `ui/tab_browsers.py`）

| 功能 | 说明 | 迁移方式 |
|:-----|:-----|:-----|
| 浏览器规则列表 | 域名→分类映射 | JS 渲染 + 增删改 |
| 新增规则 | 输入框 + 下拉选择分类 | 表单提交 |

### 2.4 游戏目录（原 `ui/tab_games.py`）

| 功能 | 说明 | 迁移方式 |
|:-----|:-----|:-----|
| 游戏进程路径列表 | 扫描到的游戏 exe 路径 | JS 列表渲染 |
| 添加游戏目录 | 文件夹选择器 | 后端执行目录扫描，返回结果 |
| 启用/禁用游戏追踪 | Toggle | 配置开关 |

### 2.5 忽略名单（原 `ui/tab_ignore.py`）

| 功能 | 说明 | 迁移方式 |
|:-----|:-----|:-----|
| 忽略应用列表 | 列表展示 | JS 渲染 `/api/ignore` |
| 添加到忽略 | 进程选择器 | 网页端进程列表 |
| 从忽略恢复 | 删除操作 | Fetch DELETE |

### 2.6 数据库管理（原 `ui/tab_database.py`）

| 功能 | 说明 | 迁移方式 |
|:-----|:-----|:-----|
| 数据库状态 | 文件大小、记录数 | `/api/database` GET |
| 备份数据库 | 触发备份 | Fetch POST |
| 清理旧数据 | 按天数清理历史 | 输入框 + 确认弹窗 |
| 导出数据 | CSV/JSON 导出 | 后端生成，前端触发下载 |

### 2.7 崩溃日志与反馈（原 `ui/tab_feedback.py`）

| 功能 | 说明 | 迁移方式 |
|:-----|:-----|:-----|
| 查看崩溃日志 | 日志文件内容展示 | Fetch `/api/feedback/logs` |
| 导出反馈包 | 打包日志+配置+数据库摘要 | Fetch `/api/feedback/export` |
| 提交反馈 | 表单（描述+联系方式） | Fetch POST `/api/feedback` |

## 三、网页视觉设计

### 3.1 主题体系（2 套）

#### 主题一：童话 🏰

> 延续原神童话风报告的设计语言，用于设置界面。

| 元素 | 设计方向 |
|:-----|:-----|
| 整体风格 | 温暖、柔和、有故事感 |
| 背景 | 浅色渐变（米白 → 淡粉/淡蓝），可带微弱纹理 |
| 卡片 | 圆角大（16px+），轻微阴影，白色/暖白底色 |
| 主色调 | 森林绿 `#4CAF50` / 天空蓝 `#87CEEB` |
| 强调色 | 暖橙 `#FF9800` / 粉红 `#E91E63` |
| 字体 | 圆润无衬线（系统默认优先） |
| 图标 | 拟物化/手绘风格（emoji 或 SVG 图标库） |
| 动画 | 轻微弹性过渡（cubic-bezier 自定义） |
| 适用人群 | 普通用户、喜欢轻松风格的用户 |

#### 主题二：极客 🖥️

> 参考 v0.2.0 精美模式的暗色设计，但更强调"技术感"。

| 元素 | 设计方向 |
|:-----|:-----|
| 整体风格 | 酷炫、高密度信息、专业感 |
| 背景 | 深色（`#0a0f1a` → `#141b2d` 渐变），参考 v0.2.0 配色 |
| 卡片 | 半透明深色背景（`rgba(26,35,50,0.8)`），细边框 `#2a3a4e` |
| 主色调 | 亮蓝 `#1E90FF` / 青绿 `#00CED1` |
| 强调色 | 霓虹粉 `#FF00FF` / 萤黄 `#FFFF00`（用于告警/高亮） |
| 字体 | 等宽字体优先（Consolas / Fira Code） |
| 图标 | 线性图标（SVG stroke 风格） |
| 动画 | 快速精准过渡，发光效果（box-shadow glow） |
| 终端感元素 | 可选：设置项带 `>_` 前缀、版本号用 `v0.3.0` 等宽显示 |
| 适用人群 | 开发者、技术用户、喜欢暗色主题的用户 |

### 3.2 主题切换

- 设置页面右上角固定主题切换按钮（🌸 童话 / 🖥️ 极客）
- 主题偏好保存在 `config.json → web_theme: "fairy" | "geek"`
- 切换主题后页面无刷新替换 CSS 文件（` <link rel="stylesheet">` href 替换）

### 3.3 布局结构（两套主题共用）

```

│  Header: UsageTracker v0.3.0    [🌸/🖥️ 主题]  │
├──────────┬──────────────────────────────────────────┤
│          │                                          │

## 四、实施顺序

```

  ├─ 新建 ui/web/ 目录结构
  ├─ 扩展 bridge_handler.py，新增 /settings 和 API 端点
  ├─ 实现 config.json 的读写 API（/api/config）
  └─ 编写 index.html 基础布局 + 两套主题 CSS

Phase 2: 通用标签页
  ├─ 应用信息展示（版本号、许可）
  ├─ 语言切换
  ├─ 开机自启 / 自动报告开关
  ├─ 检测模式选择
  └─ 检查更新按钮

Phase 3: 分类管理
  ├─ 分类列表展示（JS 渲染）
  ├─ 新增/删除分类
  ├─ 添加应用到分类（进程选择器网页版）
  └─ 分类建议面板

Phase 4: 浏览器 + 游戏 + 忽略
  ├─ 浏览器规则管理
  ├─ 游戏目录管理
  ├─ 忽略名单管理
  └─ 进程选择器组件复用

Phase 5: 数据库 + 反馈
  ├─ 数据库状态展示 + 备份/清理
  ├─ 崩溃日志查看
  ├─ 导出反馈包
  └─ 提交反馈表单

Phase 6: 首次引导（Onboarding）
  ├─ 浏览器打开引导页面（替代 tkinter onboarding）
  ├─ 强调开机自启建议
  ├─ UI 模式说明（网页端是唯一设置界面）

  ├─ 移除 ui/settings_window.py（tkinter 设置窗口）
  ├─ 托盘右键菜单"设置"改为打开浏览器
  ├─ main.py 检测 first_run 后打开浏览器引导页
  └─ 更新 README 说明设置方式变更

Phase 8: 测试 + 打包发版
  ├─ 功能测试（两套主题各测试一遍）
  ├─ 浏览器兼容性测试（Edge/Chrome/Firefox）
  ├─ PyInstaller 打包（确保 ui/web/ 资源被打包）
  ├─ Inno Setup 更新（安装后 first_run 写入）
  └─ GitHub Release v0.3.0

## 五、不在 v0.3.0 范围内

| 功能 | 预估版本 | 推迟原因 |
|:-----|:-----|:-----|
| 浏览器域名精准追踪（CDP） | v0.4.0 | 技术复杂度高，用户门槛高 |
| 应用图标真实提取（exe） | v0.4.0 | 需要 PowerShell + 图标提取库 |
| 数据库加密 | v0.4.0 | 需要 SQLCipher，增加依赖 |
| 云同步 | v1.0.0 | 需要后端服务 |
| 插件系统 | v1.2.0 | 需要稳定的 API 设计 |
| 移动端适配（网页端响应式） | v0.3.x | 优先级低，桌面端为主 |

## 六、风险与注意事项

| 风险 | 应对措施 |
|:-----|:-----|
| 浏览器打开体验不如原生窗口 | 用 `--app` 模式打开（Edge/Chrome PWA），接近原生体验 |
| Bridge 服务端口冲突 | 端口 19234 已被占用时自动+1 重试，配置可修改 |
| 网页端需要浏览器支持 | 检测系统默认浏览器，不支持时降级提示 |
| 第一次使用需要打开浏览器（用户不习惯） | Onboarding 明确引导，托盘菜单"设置"直接打开 |
| 两套主题维护成本 | 提取 common.css 共用变量，主题 CSS 仅覆盖变量 |
| 进程选择器需要后端扫描 | `/api/processes` 端点，后端 psutil 扫描，前端搜索过滤 |
| 极客主题"终端感"设计细节待定 | v0.3.0 规划阶段先定大方向，实施前细化设计稿 |

## 七、关键决策记录

- **GUI 完全弃用**：v0.3.0 起，`ui/settings_window.py` 不再维护，设置全部改为网页端
- **主题允许用户切换**：`config.json → web_theme` 字段，默认"童话"（更大众）
- **Bridge 服务常驻**：追踪器运行时网页端始终可访问，关闭浏览器不影响追踪
- **端口固定**：默认 `127.0.0.1:19234`，与现有 Bridge 报告通信复用（报告 HTTP 服务已占 19234，需合并端点）
