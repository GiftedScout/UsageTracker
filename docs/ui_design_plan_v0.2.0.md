# UsageTracker v0.2.0 设置界面 UI 设计方案

## 一、设计目标
1. 提供两套界面模式：精简模式（保留当前 tkinter 原生控件）和精美模式（新增）。
2. 精美模式参考 `assets/banner.png` 封面设计风格，作为安装后首次打开引导界面的默认风格。
3. 两套模式共用同一套 `ConfigManager`，仅 UI 层不同。
4. 用户可在设置界面右上角切换模式，切换后重启设置窗口生效。

## 二、精美模式界面规范

### 2.1 风格参照
- 主要参考 `assets/banner.png` 的色彩方案、圆角风格和卡片布局。
- 建议从 banner.png 提取：
  - 主色调（用于标题、按钮等）
  - 次色调（用于卡片背景、输入框等）
  - 强调色（用于开关选中状态、图标等）
  - 圆角半径（建议 8-12px）
  - 阴影效果（轻微投射，提升层次感）

### 2.2 布局结构（参考 roadmap-v0.2.0.md 的 ASCII art）


### 2.3 详细组件说明

#### 头部区域
- 左侧：应用 logo（来自 assets/logo.png）+ 应用名称 "UsageTracker"
- 右侧：模式切换按钮（文字显示当前模式的反向，如“精简模式”表示当前是精美模式，点击切换到精简模式）
- 整头部背景色使用 banner.png 的主色调，文字为白色或浅色以确保可读性

#### 主体区域
- 采用卡片式布局，每个功能区域（通用、游戏/浏览器/分类等）放在一个圆角卡片内
- 卡片之间有适当间距（建议 16px）
- 卡片内部使用较浅的背景色（次色调或白色透明度），内边距 20px
- 标题使用较大字体和主色调
- 控件（复选框、下拉框、开关）采用自定义样式，保持与 banner.png 一致的圆角和颜色

#### 底部按钮区域
- 两个按钮：保存并关闭（主按钮，使用主色调背景，白色文字）和取消（次按钮，透明背景，主色调边框和文字）
- 按钮圆角与卡片一致，带有悬停效果

### 2.4 Tab 切换实现
- 方案一：横向 Tab（类似当前 Notebook 但自绘样式）
- 方案二：竖向侧栏（左侧图标列表，点击切换右侧内容）
- 建议先实现横向 Tab，后续根据用户反馈考虑侧栏
- Tab 自绘需要：
  - 选中状态：使用主色调填充，文字白色
  - 未选中状态：透明或次色调背景，文字为深色
  - 悬停状态：轻微背景色变化
  - Tab 之间有分隔线或间距

## 三、实现方案

### 3.1 新建文件
1. `ui/styled_widgets.py`：封装自定义风格控件
   - `StyledFrame`: 带圆角和背景色的 Frame
   - `StyledButton`: 带悬停效果的 Button
   - `StyledLabel`: 颜色和字体可配置的 Label
   - `Card`: 由 `StyledFrame` 组成的卡片容器，内带标题和内容区
   - `StyledNotebook`: 自绘的 Notebook（用于横向 Tab）

2. `ui/onboarding.py`：安装后首次引导界面逻辑
   - 使用精美模式显示欢迎卡片
   - 引导用户完成初始设置
   - 用户点击"开始追踪"后设置 `first_run=false` 并进入正常追踪模式

### 3.2 现有文件修改
1. `ui/settings_window.py`：
   - 添加 `_ui_mode` 属性，从 config 读取 `ui_mode`（默认 rich）
   - 根据 `_ui_mode` 选择使用原生 tkinter 控件（简单模式）或 styled_widgets（精美模式）
   - 添加模式切换按钮的回调函数：切换 `ui_mode` 配置，重启设置窗口
   - 精美模式下使用 `Card` 和 `StyledNotebook` 重构布局

2. `src/config_manager.py`：
   - 在 `DEFAULT_CONFIG` 中添加：
     ```python
     'ui_mode': 'rich',   # 'simple' 或 'rich'
     'first_run': True,   # 安装后首次运行标记
     ```
   - 添加对应的 property 方法和验证（`ui_mode` 只允许 'simple' 或 'rich'）

3. `src/main.py`：
   - 在初始化时检查 `first_run`：
     - 如果为 True：启动精美模式设置界面（通过 `onboarding.py`）
     - 如果为 False：根据 `ui_mode` 决定启动哪种模式的设置界面（或直接进入追踪模式？根据需求，设置界面只在用户主动打开时显示，但首次运行要显示引导）
   - 实际需求：安装完成后首次打开 UsageTracker 时显示引导界面（属于精美模式的一部分），用户完成设置后进入正常追踪模式（不显示设置窗口，除非用户主动打开）。
   - 因此，`main.py` 只需在首次运行时显示引导界面，非首次运行时不自动显示设置窗口（只在用户通过托盘菜单打开设置时显示）。

4. `src/tray_app.py`：
   - 打开设置界面时，根据 `ui_mode` 决定打开哪种模式（但根据需求，设置界面应始终使用当前配置的 `ui_mode`，而不受首次运行影响）

5. `installer.iss`：
   - 安装后写入 `config.json` 中的 `first_run=true`（可以在安装完成后通过脚本修改，或在首次运行时由程序自动检测配置文件是否存在来决定）

### 3.3 降级策略
- 精美模式渲染时若发生异常（如图片加载失败、自绘控件错误），捕获异常并自动切换到精简模式
- 同时记录日志，并在下次启动时尝试再次使用精美模式（除非用户手动切换到精简模式）

## 五、实施步骤

1. 提取 banner.png 的色彩方案和圆角参数（可使用图片处理工具或手动设定初始值，后续优化）。
2. 实现 `ui/styled_widgets.py` 中的基础控件。
3. 修改 `src/config_manager.py` 添加新配置项。
4. 修改 `ui/settings_window.py` 添加模式检测和两套 UI 实现（先保留简单模式不变，再实现精美模式）。
5. 实现模式切换按钮和重启逻辑。
6. 创建 `ui/onboarding.py` 并修改 `src/main.py` 添加首次运行检测。
7. 更新 `installer.iss` 确保安装后写入 `first_run=true`。
8. 测试两种模式切换、首次运行引导以及降级策略。
9. 根据测试结果调整细节（如间距、圆角、颜色等）。

## 六、风险与注意事项

- 风险：精美模式自绘控件可能引入性能问题（尤其在老旧机器上）。
  - 应对：使用 Canvas 的 `create_rectangle` 等原始绘图，避免过度复杂的路径；绘制频率采用防抖（如 50ms 间隔）。
- 风险：色彩方案从 banner.png 提取不准确导致视觉不协调。
  - 应对：先定义一套基础色彩（基于 banner.png 主观感），后续可通过配置文件微调。
- 风险：用户习惯变化导致切换混淆。
  - 应对：在设置界面明显位置提供模式说明和切换按钮，使用图标+文字双重提示。
